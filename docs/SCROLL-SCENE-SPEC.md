# «One platform» — Scroll Scene Spec (pinned, Phase D contract)

> Источник: клиентский спек (Олег, голосом, 2026-07-02) + Figma `19:215` (волна 1+2
> собраны), `41:4265` (состояние сцены), `20:819` (финал с текстом и чипсами).
> Референс поведения: **https://www.branch.computer/** — карточки влетают с чуть
> разной (рандомизированной) скоростью; «это даёт очень гибкий эффект». Без blur.

## Ключевое架构ное решение

**Constellation (19:215) и PlatformIntro (20:819) — НЕ две секции, а ОДНА pinned
scroll-сцена.** Экран пиннится, скролл скрабит таймлайн. Статическая версия
(без JS / reduced-motion / до Phase D) — две секции подряд, как сейчас собрано.

## Хореография (по скроллу, скраб)

0. **Пустота.** Тёмный экран (канва `--bg`), ничего нет.
1. **Печатается заголовок, часть 1:** «One platform» → «for review management»
   (появление слов — как набор/последовательное проявление).
2. **Влетает волна 1 — ТЁМНЫЕ пиллы** (bg `--surface-pill` rgba(0,0,0,.5) —
   подтверждено пиксель-сэмплингом рендера: #080911, композит нашего токена
   #080A11, Δ=1 = AA-шум): **Google Reviews, Amazon Reviews, App Store Reviews**
   (ровно 3).
   Эффект: **fade + scale-down** — из opacity 0 и КРУПНОГО масштаба карточка
   плавно уменьшается на своё место и становится непрозрачной.
   **Рандомизация скоростей** (по branch.computer): каждая карточка со своей
   длительностью/задержкой. Blur НЕ используется.
3. **Печатается часть 2:** «and social customer service» (фиолетовая, Bold).
4. **Влетает волна 2 — ФИОЛЕТОВЫЕ пиллы** (bg `--surface-pill-social`
   rgba(171,13,255,.2) = #30124E композитом, пиксель-подтверждено): TikTok
   Comments, Instagram Comments, Facebook Comments, Messenger, Instagram DMs,
   WhatsApp, **Google Maps**, eBay, Etsy, Trustpilot (ровно 10).
   Тот же эффект (fade + scale + рандом скоростей).
   Состав волн = ЦВЕТ пилла в Figma (не смысловая категория) — проверен
   пиксель-сэмплингом каждого пилла 2026-07-02.
5. **Крутим дальше:** пиллы исчезают, **заголовок остаётся на месте**.
6. **Блок объяснения — ОДИН слитый бит** (правка клиента 2026-07-02: «сократить
   путь скролла», не «скролл ради скролла»): описание fade+снизу-вверх, и
   **сразу поверх него** три чипсы «чик-чик-чик» (stagger сохранён, плотнее) —
   без отдельного скролл-этажа для чипсов.
7. **Расцепление:** короткий settle — и обычный скролл к «Everything you need».
   Дорожка сцены: **300vh** (450 → 360 → 300, клиент+Олег 2026-07-08).
   Скраб стартует с `top bottom` — таймлайн живёт с момента, когда сцена
   показывается снизу: заголовок печатается и волна 1 прилетает, **пока первый
   экран ещё уматывается** (никакой мёртвой черноты между экранами). Пин
   включается на ~трети дорожки; финальный бит собран к ~95%.

## Реализация (обязательно)

- **Пин через CSS `position: sticky`** + GSAP ScrollTrigger scrub — НЕ GSAP pin
  (урок Verifly: reload-race у GSAP pin, мигрировали на sticky).
- Карточки: transform/opacity only (scale ~1.6→1.0, opacity 0→1), стагер с
  рандом-сидом на индекс (детерминированный рандом — чтобы не прыгал между
  загрузками).
- Заголовок: разбивка на слова для «печати» (part 1, потом part 2).
- `prefers-reduced-motion` → статика: два экрана без пина (текущая сборка).
- Мобайл: поведение уточнится с мобильными фреймами (ожидаются от Олега).

  этому неймингу.
